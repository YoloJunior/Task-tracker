Захист проекту: Веб-додаток реєстрації на Vue.js 3

Про що проект

Це форма реєстрації для музичної платформи "Maggot". Основна ідея - зробити зручний інтерфейс для реєстрації різних типів користувачів: фанатів музики, музикантів та організаторів подій. Проект написаний на Vue.js 3 з використанням сучасного Composition API.


Які технології використовував

Vue.js 3 (версія 3.2.13) - основний фреймворк для побудови інтерфейсу. Обрав саме третю версію, бо там є Composition API, який дозволяє краще організувати код.

IMask (версія 7.6.1) - бібліотека для маскування вводу. Використовую її для телефонного номера, щоб автоматично форматувався в +38(0XX)-XXX-XX-XX.

FontAwesome 6.5.1 - іконки для полів вводу. Підключив через CDN в index.html.

Vue CLI 5 - для збірки проекту та локального сервера розробки.

CSS3 - стилізація без фреймворків, все писав вручну. Використав Flexbox, Grid, анімації.


Структура проекту

web/
  public/
    index.html - головний HTML файл, тут підключаю FontAwesome
  src/

    main.js - точка входу, створення Vue додатку
    components/ - папка для компонентів

Що робить додаток

Форма збирає такі дані від користувача:
- Email для входу в систему
- Пароль (мінімум 6 символів)
- Ім'я та прізвище
- Дата народження
- Роль (фанат, музикант або організатор)
- Номер телефону з українським форматом
- Стать

Всі поля провалідовані. Якщо щось не так - під полем з'являється червоне повідомлення про помилку.


Як реалізована валідація

Створив реактивний об'єкт errors, куди записуються помилки для кожного поля. Функція validateForm() перевіряє всі поля перед відправкою:

- Email перевіряю регулярним виразом на наявність @ та домену
- Пароль - просто довжина мінімум 6 символів
- Ім'я і прізвище - мінімум 2 символи
- Телефон - має бути рівно 12 цифр (без урахування маски)
- Інші поля просто перевіряю чи вони не порожні

Помилки показуються в реальному часі при введенні, плюс повна перевірка при натисканні кнопки.


Як працює Composition API

Замість старого Options API (data, methods, computed) використовую setup синтаксис:

const formData = reactive({
  email: '',
  password: '',
  firstName: '',
  ...
})

const errors = reactive({
  email: '',
  password: '',
  ...
})

Функція reactive() робить об'єкт реактивним - коли змінюється значення, автоматично оновлюється інтерфейс.


Маска для телефону

В onMounted() ініціалізую IMask на полі телефону:

onMounted(() => {
  const phoneInput = document.getElementById('phone')
  phoneMask = IMask(phoneInput, {
    mask: '+38(000)-000-00-00'
  })
})

Тепер при введенні цифр вони автоматично форматуються.


Стилізація

Зробив темну тему з помаранчевими акцентами. Всі кольори винесені в CSS змінні:

--primary: #f97316 (помаранчевий)
--bg-dark: #050710 (темний фон)
--bg-card: #111827 (фон картки)
--text-main: #f3f4f6 (білий текст)
--danger: #ef4444 (червоний для помилок)

Додав ефект glassmorphism - це коли фон картки трохи розмитий (backdrop-filter: blur). Виглядає сучасно.

Є дві анімації:
- fadeInUp - форма плавно з'являється знизу при завантаженні
- fadeInScale - повідомлення про помилки плавно з'являються

Адаптивність зробив через медіа запит - на екранах менше 600px форма перебудовується в одну колонку замість двох.


Що показати на демонстрації

1. Заповнити форму правильно і відправити - покаже alert про успішну реєстрацію
2. Спробувати відправити з порожніми полями - з'являться всі помилки
3. Ввести кривий email без собаки - покаже помилку валідації
4. Почати вводити телефон - видно як працює маска
5. Зменшити вікно браузера - форма стане в одну колонку
6. Можна відкрити консоль і показати що дані виводяться туди при відправці

Відповіді на можливі питання

Чому Vue 3, а не React?
Vue простіший для вивчення, має хорошу документацію українською, і Composition API дає таку ж гнучкість як React hooks.

Чому не взяв готовий UI фреймворк типу Vuetify?
Хотів показати що вмію писати CSS сам, плюс так менше залежностей і повний контроль над виглядом.

Як працює реактивність?
Vue 3 використовує JavaScript Proxy щоб відслідковувати зміни в об'єктах. Коли щось змінюється - Vue автоматично перерендерює потрібні частини DOM.

Куди йдуть дані при відправці?
Поки що просто в console.log. В реальному проекті робив би fetch або axios запит на API.

Що таке SPA?
Single Page Application - вся сторінка завантажується один раз, далі контент змінюється динамічно без перезавантаження.
